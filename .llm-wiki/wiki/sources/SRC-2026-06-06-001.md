# project description.md

## Summary

Notifier is a standalone service that acts as a unified dispatch layer for sending emails and push notifications on behalf of other services. Rather than having each automated process communicate directly with a mail server (which requires individual authorization), Notifier becomes the single source of truth — services send requests to Notifier, and it handles the rest of the delivery.

Authentication is API-key based. Each consumer (service/process) receives an API key and is assigned a unique sender email address (e.g., automater-noreply@domain.com). The system is required to implement abuse detection, anti-spam, and DDoS protection mechanisms. It must also enforce rate limiting and provide job status visibility to consumers for audit and protection purposes.

## Key Takeaways

- Notifier is a standalone service that unifies email and push notification dispatch for enterprise services.
- It replaces direct mail server integration by acting as a single source of truth for all outgoing notifications.
- Authentication is API-key based, with each consumer getting a unique sender email address.
- The system must implement abuse detection, anti-spam, DDoS protection, rate limiting, and job status tracking.
